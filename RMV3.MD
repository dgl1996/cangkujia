### 项目名称
**仓酷家** - 中小仓库的美图秀秀

仓库户型图 + 3D物流对象资产 = 快速输出3D物流仓库效果图

5分钟拖拽输出3D仓库规划效果图和实施清单，不用学CAD，不用等规划部出图，人人都是物流规划师！

## 技术架构

### 前端 (Frontend)
- **框架**: Vue 3 + Vite
- **3D引擎**: Three.js
- **UI组件**: 自定义组件 + Element Plus
- **端口**: 5173

### 后端 (Backend)
- **框架**: FastAPI
- **语言**: Python
- **端口**: 8000

### 3D模型系统
- **格式**: GLB/GLTF
- **数量**: 40个模型文件
- **位置**: `/frontend/public/assets/models/`
- **生成工具**: Python + Trimesh
- **自动添加工具**: `scripts/model_generator_template.py` (正式命名: **3D对象自动添加工具**)

### 坐标系与单位
- **Three.js**: Y轴向上，单位cm
- **Trimesh**: Z轴向上，单位mm
- **转换**: -90度绕X轴旋转，缩放0.1

### 重要教训：3D模型单位一致性
**问题**: 新模型与旧模型单位不一致导致显示异常
**原因**: 
- 旧模型使用mm单位，ThreeScene.vue中缩放0.1
- 新模型错误转换为cm单位，导致尺寸缩小10倍
**解决**: 
- 所有模型统一使用mm单位生成
- ThreeScene.vue中统一使用0.1缩放
- 缩放后再计算边界框(box.min.y)调整位置
**检查点**: 
- [ ] 生成模型前确认单位是mm
- [ ] 检查缩放比例是否为0.1
- [ ] 验证模型加载后的尺寸输出

### 自定义货架功能方案 (2026-03-13更新)

**方案变更**: 从"P1页直接点击放置"改为"P2页创建→P1页拖拽"

**原因**: 
- 原方案（P1页点击放置）存在技术难题：模型ID匹配、边界框计算、射线检测等问题
- 新方案复用已验证的拖拽机制，稳定性更高

**新方案流程**:
1. **P2页（ModelLibrary.vue）**: 用户点击"自定义货架" → 两层级联选择 → 添加到"我的模型"
2. **数据同步**: 保存到 localStorage (`myModels` key)，P1/P2页自动同步
3. **P1页（CoreFunction.vue）**: 展开"👤 我的模型"分类 → 拖拽到画布放置

**技术实现**:
- **40个货架模型预生成**: 使用 `scripts/generate_all_shelves.py` 批量生成
  - 轻型货架: 10个（A15-4/5, A20-4/5/6 单组+配组）
  - 中型货架: 6个（B20-4/5/6 单组+配组）
  - 高位货架: 24个（C23/25/27 系列 3/4/5/6层 单组+配组）
- **元数据管理**: `shelf-models-metadata.json` 存储所有规格参数
- **两层级联选择器**: 第一层选类型（轻/中/高），第二层选规格
- **已预置模型禁用**: 6个预置模型在第二层显示"已预置"并禁用选择
- **模型预加载**: `ThreeScene.vue` 的 `modelFileMap` 包含全部40个模型

**UI设计**:
- 弹窗分上下两区：上层选择器并排，下层3D预览(左大) + 规格信息(右小)
- 规格显示：承重、层高、适配净高
- 3D预览：实时显示选中模型，支持自动旋转

**相关文件**:
- `scripts/generate_all_shelves.py` - 批量生成40个GLB模型
- `frontend/public/assets/shelf-models-metadata.json` - 模型元数据
- `ModelLibrary.vue` - P2页自定义货架弹窗、两层级联选择器
- `CoreFunction.vue` - P1页"我的模型"显示、拖拽逻辑
- `ThreeScene.vue` - 模型预加载、放置逻辑

**Bug修复记录**:
- ✅ **重复声明错误**: `selectedModel` 变量重复声明，已删除重复项
- ✅ **布局错乱**: `.preview-body` 默认grid布局覆盖为flex纵向布局
- ✅ **模型只有一个点**: `modelFileMap` 未包含新模型，已添加全部40个

**状态**: ✅ 已完成并验证

**Bug修复：选中状态被清空问题**

**问题现象**
文字标注点击选中后，右侧面板需要按住鼠标左键才能显示，松开鼠标后面板立即消失。

**根因分析**
事件机制冲突：
1. `mousedown` → `mouseup` → `click` 事件顺序
2. `onTextLabelMouseDown` 中启动拖动，使用 `window.addEventListener('mouseup', onMouseUp)`
3. `onCanvasClick` 使用 Vue 的 `@click`，与 `window` 上的 `mouseup` 是独立的监听机制
4. `stopPropagation()` 只能阻止冒泡到父元素，不能阻止独立的 `click` 事件

**修复方案：标志位拦截**
```javascript
// 添加标志位
const ignoreNextCanvasClick = ref(false);

// 文字标注 mousedown 处理
function onTextLabelMouseDown(event, labelId) {
  ignoreNextCanvasClick.value = true;  // 标记：接下来忽略画布click
  selectedTextLabel.value = labelId;
  // ... 其他逻辑
}

// 画布 click 处理
function onCanvasClick(event) {
  if (ignoreNextCanvasClick.value) {
    ignoreNextCanvasClick.value = false;  // 重置标志
    return;  // 不执行清空选中
  }
  // ... 原有逻辑
}
```

**修复后行为**
1. 点击文字标注 → 文字变橙色，显示虚线边框，右侧面板显示属性和操作按钮
2. 松开鼠标 → 选中状态保持，面板不消失

### 2026-03-16: 批量复制旋转后货架重叠Bug修复

**问题描述**
货架旋转90度后，进行批量复制（如3行2列，行距3米，列距0），中型、轻型货架出现重叠，而高位货架正常。

**根因分析**
批量复制位置计算使用了**世界坐标系尺寸**：
```javascript
const rowOffset = r * (getObjectDepth(originalObj) + rowSpacing * 100);
const colOffset = c * (getObjectWidth(originalObj) + colSpacing * 100);
```

当货架旋转90度后：
- 世界坐标系的X尺寸 = 原货架的Z尺寸（长度）
- 世界坐标系的Z尺寸 = 原货架的X尺寸（宽度）

导致行/列间距计算错误，货架重叠。

**修复方案：使用本地尺寸**

**1. 新增获取本地尺寸的方法**（不考虑旋转）：
```javascript
// 获取对象本地宽度（模型本身的X尺寸）
function getObjectLocalWidth(obj) {
  const clonedObj = obj.clone();
  clonedObj.rotation.set(0, 0, 0);
  const box = new THREE.Box3().setFromObject(clonedObj);
  return box.max.x - box.min.x;
}

// 获取对象本地深度（模型本身的Z尺寸）
function getObjectLocalDepth(obj) {
  const clonedObj = obj.clone();
  clonedObj.rotation.set(0, 0, 0);
  const box = new THREE.Box3().setFromObject(clonedObj);
  return box.max.z - box.min.z;
}
```

**2. 修改批量复制位置计算**：
```javascript
// 使用本地尺寸，不受旋转影响
const rowOffset = r * (getObjectLocalDepth(originalObj) + rowSpacing * 100);
const colOffset = c * (getObjectLocalWidth(originalObj) + colSpacing * 100);
```

**修复后行为**
- 无论货架旋转多少度（0°、90°、180°、270°），批量复制都使用货架的**真实本地尺寸**
- 行间距基于货架本地深度（前后方向）
- 列间距基于货架本地宽度（左右方向）
- 轻型、中型、高位货架旋转后都能正确排列，不会重叠

**影响文件**
- `frontend/src/components/3d/ThreeScene.vue` - 批量复制位置计算逻辑、新增本地尺寸获取方法

### 2026-03-19: 对齐线功能完整实现

**功能概述**
实现对齐线功能，帮助用户精确放置3D对象，类似CAD的辅助线功能。

**核心功能**
1. **对齐线绘制**
   - 点击工具栏"➕对齐线"按钮进入绘制模式
   - 按钮变为红色激活状态（背景、边框、文字、阴影）
   - 在3D场景中点击两点确定直线方向
   - 生成红色无限长虚线（向两端延伸1000米）
   - 再次点击按钮或按ESC退出绘制模式
   - 自动切换到选择模式

2. **对齐线管理**
   - 左侧面板"对齐线"卡片显示对齐线列表
   - 列表项显示对齐线名称（如"对齐线1"）
   - 点击列表项：该项显示红色边框和背景高亮
   - 3D场景中对应的对齐线变为黄色高亮
   - 支持删除单条对齐线
   - 支持清空全部对齐线（带确认对话框）

3. **技术实现**
   - **颜色**: 红色 `0xff0000`，醒目对比
   - **材质**: `LineDashedMaterial`，虚线效果
   - **长度**: 无限长直线（向两端延伸100000cm）
   - **高度**: 基于 `warehouseConfig.baseHeight`，在地面之上2cm
   - **相机**: 俯视仓库中心（高度10000cm），确保对齐线可见
   - **保存**: 对齐线数据随项目一起保存和加载
