# 仓酷家开发军规（Trae必须遵守）

## 方案设计原则（每次修改前必须执行）
1. **影响范围检查**：修改前必须回答"会影响哪些其他功能？"
2. **副作用评估**：必须列出至少1个潜在风险和1个预防措施  
3. **回退方案**：如果改坏了，如何快速恢复原状？
4. **最小改动**：能用配置解决，不动代码；能改一行，不改十行

## 禁止事项（红线）
- 禁止只说"修复了XX"而不说"影响了YY"
- 禁止删除看似"没用"的代码（可能别的功能依赖它）
- 禁止修改全局样式（可能破坏其他页面）
- 禁止修改公共工具函数而不检查谁在用

## 输出格式（必须按此回复）
每次提供方案时，必须包含：

### 1. 改动说明（1句话）
【要改哪里，怎么改】

### 2. 影响范围（必须回答）
- **直接影响**：当前这个功能本身
- **潜在影响**：可能波及的3个其他功能（列出具体文件名或功能名）
- **无影响**：确认不会动的区域

### 3. 风险评估（必须回答）
- **低风险**：只影响当前页面，改坏了 reload 就恢复
- **中风险**：影响多个组件，需要测试A、B、C功能
- **高风险**：改动核心文件，可能导致系统崩溃

### 4. 回退方案（必须回答）
如果改完出现问题，如何快速回退：
- 方法1：Git撤销（具体命令）
- 方法2：注释掉新代码，恢复原代码（具体行号）
- 方法3：还原备份文件（备份路径）

### 5. 测试清单（勾选框形式）
改完后必须验证：
- [ ] 当前功能正常
- [ ] 功能A正常（列出具体测试步骤）
- [ ] 功能B正常

## 特别警告（Three.js相关）
- 修改材质/几何体前：确认是否共享引用（影响其他对象）
- 修改scene对象前：确认是否会影响渲染循环
- 添加事件监听前：确认是否重复添加（内存泄漏）
