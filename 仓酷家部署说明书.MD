
## 📝 五、问题与解决方案记录

### 5.1 已解决问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| SSH连接问题 | 首次连接需要确认主机密钥 | 使用 `-o StrictHostKeyChecking=no` 参数 |
| Nginx未安装 | 服务器未预装Nginx | `yum install -y nginx-core` |
| 403 Forbidden | 目录权限不足 | `chmod -R 755 /var/www/cangkujia/frontend/dist/` |
| 证书上传失败 | SSL目录不存在 | `mkdir -p /etc/nginx/ssl` |
| 静态文件无法访问 | 所有者不正确 | `chown -R nginx:nginx /var/www/cangkujia/frontend/dist/` |
| 端口8000被占用 | 之前的服务进程未清理 | `fuser -k 8000/tcp` 杀掉占用进程 |
| 服务启动失败 | systemd服务文件未创建 | 手动创建 `/etc/systemd/system/cangkujia.service` |
| Nginx配置语法错误 | return语句使用了反引号 | 删除反引号，使用标准语法 |
| API 404错误 | HTTP配置中缺少API反向代理 | 在HTTP server中添加location /api/配置 |
| 301重定向循环 | API请求被重定向到HTTPS | 在HTTP配置中优先匹配API路径 |
| PostgreSQL数据库连接失败 | Supabase只提供IPv6地址，服务器无法访问 | 临时切换到SQLite本地数据库进行测试 |

### 5.1 已解决问题（续）

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 3D模型无法加载 | COS跨域访问未配置 | 在COS控制台配置CORS规则，允许https://cangkujia666.com访问 |
| 3D模型拖拽无效 | 浏览器CORS策略阻止 | 配置COS CORS规则，添加GET/HEAD方法，设置Max-Age为3600 |
| Node.js版本过低 | 服务器默认Node.js 18，Vite需要20+ | 使用nvm安装Node.js 20.20.2 |
| 前端构建失败 | node_modules版本不兼容 | 清理后重新安装：rm -rf node_modules && npm install |
| 部署目录为空 | 构建输出在temp-repo目录 | 从/var/www/cangkujia/temp-repo/frontend/dist复制 |
| Nginx 403错误 | 默认server配置冲突 | 修改/etc/nginx/nginx.conf，更新root目录 |
| HTTPS无法访问 | 443端口未监听 | 在主配置文件中添加HTTPS server块 |
| 证书域名不匹配 | 证书为cangkujia666.com，用IP访问 | 使用域名访问https://www.cangkujia666.com/ |
