### 4.3 支付流程

```
用户点击"升级专业版"
    ↓
调用 /api/payment/create-order
    ↓
微信支付Native API生成二维码
    ↓
返回 code_url 给前端显示二维码
    ↓
用户扫码支付
    ↓
微信发送回调到 /api/payment/callback
    ↓
解密回调数据
    ↓
更新订单状态为"paid"
    ↓
激活用户专业版权限
    ↓
返回 SUCCESS 给微信
```

### 4.4 微信支付API v3签名

**签名算法**: RSA-SHA256

**Authorization格式**:
```
WECHATPAY2-SHA256-RSA2048 mchid="{商户号}",nonce_str="{随机字符串}",signature="{签名}",timestamp="{时间戳}",serial_no="{证书序列号}"
```

**签名计算**:
```
签名字符串 = HTTP方法 + "\n" + URL路径 + "\n" + 时间戳 + "\n" + 随机字符串 + "\n" + 请求体 + "\n"
签名 = RSA-SHA256(签名字符串, 私钥).toString("base64")
```

### 5.2 详细问题记录

#### 问题1: API路由404错误

**问题描述**:
用户点击"升级专业版"后，支付弹窗显示"用户不存在"错误，无法显示微信支付二维码。

**根本原因**:
1. **Turbopack缓存损坏**: Next.js 16默认使用Turbopack，但存在兼容性问题
2. **`createServerClient` 不兼容**: `@supabase/ssr` 的 `createServerClient` 在API路由中不兼容
3. **缺少 Authorization Header**: 前端请求未携带token

**解决方案**:

**第一步：切换到Webpack模式**
```json
// package.json
"dev": "next dev --webpack"

// next.config.ts
// 移除 turbopack 配置
```

**第二步：修复API路由**
```typescript
// 错误
import { createServerClient } from '@supabase/ssr';
const supabase = createServerClient(url, key, { cookies: {...} });

// 正确
import { createClient } from '@supabase/supabase-js';
const supabase = createClient(url, serviceRoleKey);
```

**第三步：添加Authorization header**
```typescript
// 前端请求
const token = session?.access_token;
fetch('/api/payment/create-order', {
  headers: {
    'Authorization': `Bearer ${token}`,
    'Content-Type': 'application/json'
  }
});

// 后端验证
const { data: { user } } = await supabase.auth.getUser(token);
```

#### 问题2: 登录跳转失败

**问题描述**:
用户登录成功后无法跳转到工作台页面，系统反复将用户重定向回登录页面。

**根本原因**:
Supabase Auth的cookie没有正确持久化到浏览器，导致服务端无法验证用户登录状态。

**解决方案**:

**修复proxy.ts**（Next.js 16使用proxy.ts替代middleware.ts）
```typescript
import { createServerClient } from '@supabase/ssr';
import { NextResponse } from 'next/server';

export async function GET(request: Request) {
  const supabaseResponse = NextResponse.next();
  const supabase = createServerClient(url, key, {
    cookies: {
      getAll() { return request.cookies.getAll(); },
      setAll(cookiesToSet) {
        cookiesToSet.forEach(({ name, value }) => request.cookies.set(name, value));
        supabaseResponse.cookies.setAll(cookiesToSet);
      }
    }
  });

  const { data: { user } } = await supabase.auth.getUser();

  if (!user) {
    return NextResponse.redirect(new URL('/login', request.url));
  }

  return supabaseResponse;
}
```

**修复login/page.tsx**
```typescript
// 使用router.push + router.refresh
const handleLogin = async () => {
  const { error } = await signInWithPassword(email, password);
  if (!error) {
    router.push('/app');
    router.refresh(); // 强制触发服务端重新验证
  }
};
```

#### 问题3: 微信回调未到达

**问题描述**:
用户支付成功后，微信没有发送回调请求到服务器，订单状态未更新。

**根本原因**:
微信商户平台的APIv3密钥未设置，导致微信无法加密回调数据，因此不发送回调。

**解决方案**:

1. **登录微信商户平台** → API安全 → 设置APIv3密钥
2. **生成32位随机字符串**作为密钥
3. **将密钥添加到Vercel环境变量** `WECHAT_APIV3_KEY`
4. **重新部署**使环境变量生效
5. **等待5-10分钟**让配置生效
6. **使用新订单测试**（旧订单不会重发回调）

#### 问题4: 回调解密失败

**问题描述**:
微信回调已到达服务器，但解密失败，返回400错误。

**根本原因**:
nonce的编码方式错误。微信发送的nonce是原始字符串，但代码中使用了base64解码。

**错误代码**:
```typescript
// 错误：nonce是字符串，不需要base64解码
const decipher = crypto.createDecipheriv('aes-256-gcm', keyBuffer, Buffer.from(nonce, 'base64'));
```

**正确代码**:
```typescript
// 正确：nonce使用utf8编码
const nonceBuffer = Buffer.from(nonce, 'utf8');
const decipher = crypto.createDecipheriv('aes-256-gcm', keyBuffer, nonceBuffer);
```

**完整的解密函数**:
```typescript
function decryptWechatCallback(ciphertext: string, associatedData: string, nonce: string, key: string) {
  // 1. 密钥转Buffer
  const keyBuffer = Buffer.from(key, 'utf8');

  // 2. 分离认证标签和密文（ciphertext最后16字节是authTag）
  const ciphertextBuffer = Buffer.from(ciphertext, 'base64');
  const authTag = ciphertextBuffer.slice(-16);
  const encryptedData = ciphertextBuffer.slice(0, -16);

  // 3. 创建解密器
  const nonceBuffer = Buffer.from(nonce, 'utf8');
  const decipher = crypto.createDecipheriv('aes-256-gcm', keyBuffer, nonceBuffer);
  decipher.setAuthTag(authTag);
  decipher.setAAD(Buffer.from(associatedData, 'utf8'));

  // 4. 解密
  let decrypted = decipher.update(encryptedData, undefined, 'utf8');
  decrypted += decipher.final('utf8');

  return JSON.parse(decrypted);
}
```

### 7.4 关键注意事项

1. **APIv3密钥必须设置**：这是微信发送回调的前提
2. **nonce使用utf8编码**：不是base64
3. **返回SUCCESS给微信**：即使处理失败也要返回200，避免无限重试
4. **幂等性处理**：同一订单可能被多次回调，需要检查订单状态
5. **环境变量区分**：生产环境和开发环境使用不同的配置
