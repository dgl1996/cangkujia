Phase 4：Nginx配置（HTTPS）
nginx
复制
# /etc/nginx/sites-available/cangkujia
server {
    listen 443 ssl http2;
    server_name cangkujia666.com www.cangkujia666.com;
    
    # SSL证书（腾讯云免费申请）
    ssl_certificate /etc/nginx/ssl/cangkujia666.crt;
    ssl_certificate_key /etc/nginx/ssl/cangkujia666.key;
    
    # 前端Vue3（History模式支持）
    location / {
        root /var/www/cangkujia/frontend/dist;
        try_files $uri $uri/ /index.html;
        expires 7d;
    }
    
    # 后端API
    location /api/ {
        proxy_pass http://127.0.0.1:8000/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
    
    # COS模型代理（可选，解决跨域）
    location /models/ {
        proxy_pass https://cangkujia-models.cos.ap-shanghai.myqcloud.com/;
        expires 30d;  # 长期缓存
    }
}

# HTTP强制跳转HTTPS
server {
    listen 80;
    server_name cangkujia666.com www.cangkujia666.com;
    return 301 https://$server_name$request_uri;
}

⚠️ 关键优化点（2核2G生存指南）
禁止本地数据库：2G内存跑PostgreSQL会直接OOM，必须用Supabase云端
COS内网流量：服务器和COS桶都选上海地域，内网流量免费（省钱关键）
模型懒加载：Three.js不要一次性加载80个模型，按货架类型动态import
Swap分区：建议开2G Swap防止内存溢出（腾讯云轻量默认无Swap）
bash
复制
sudo fallocate -l 2G /swapfile && sudo chmod 600 /swapfile && sudo mkswap /swapfile && sudo swapon /swapfile
日志轮转：限制Nginx/Uvicorn日志大小（40G磁盘容易满）
