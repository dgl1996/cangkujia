## 技术架构

### 前端 (Frontend)
- **框架**: Vue 3 + Vite
- **3D引擎**: Three.js
- **UI组件**: 自定义组件 + Element Plus
- **端口**: 5173

### 后端 (Backend)
- **框架**: FastAPI
- **语言**: Python
- **端口**: 8000

### 3D模型系统
- **格式**: GLB/GLTF
- **数量**: 40个模型文件
- **位置**: `/frontend/public/assets/models/`
- **生成工具**: Python + Trimesh
- **自动添加工具**: `scripts/model_generator_template.py` (正式命名: **3D对象自动添加工具**)

### 坐标系与单位
- **Three.js**: Y轴向上，单位cm
- **Trimesh**: Z轴向上，单位mm
- **转换**: -90度绕X轴旋转，缩放0.1

### 重要教训：3D模型单位一致性
**问题**: 新模型与旧模型单位不一致导致显示异常
**原因**: 
- 旧模型使用mm单位，ThreeScene.vue中缩放0.1
- 新模型错误转换为cm单位，导致尺寸缩小10倍
**解决**: 
- 所有模型统一使用mm单位生成
- ThreeScene.vue中统一使用0.1缩放
- 缩放后再计算边界框(box.min.y)调整位置
**检查点**: 
- [ ] 生成模型前确认单位是mm
- [ ] 检查缩放比例是否为0.1
- [ ] 验证模型加载后的尺寸输出

### 自定义货架功能方案 (2026-03-13更新)

**方案变更**: 从"P1页直接点击放置"改为"P2页创建→P1页拖拽"

**原因**: 
- 原方案（P1页点击放置）存在技术难题：模型ID匹配、边界框计算、射线检测等问题
- 新方案复用已验证的拖拽机制，稳定性更高

**新方案流程**:
1. **P2页（ModelLibrary.vue）**: 用户点击"自定义货架" → 两层级联选择 → 添加到"我的模型"
2. **数据同步**: 保存到 localStorage (`myModels` key)，P1/P2页自动同步
3. **P1页（CoreFunction.vue）**: 展开"👤 我的模型"分类 → 拖拽到画布放置

**技术实现**:
- **40个货架模型预生成**: 使用 `scripts/generate_all_shelves.py` 批量生成
  - 轻型货架: 10个（A15-4/5, A20-4/5/6 单组+配组）
  - 中型货架: 6个（B20-4/5/6 单组+配组）
  - 高位货架: 24个（C23/25/27 系列 3/4/5/6层 单组+配组）
- **元数据管理**: `shelf-models-metadata.json` 存储所有规格参数
- **两层级联选择器**: 第一层选类型（轻/中/高），第二层选规格
- **已预置模型禁用**: 6个预置模型在第二层显示"已预置"并禁用选择
- **模型预加载**: `ThreeScene.vue` 的 `modelFileMap` 包含全部40个模型

**UI设计**:
- 弹窗分上下两区：上层选择器并排，下层3D预览(左大) + 规格信息(右小)
- 规格显示：承重、层高、适配净高
- 3D预览：实时显示选中模型，支持自动旋转

**相关文件**:
- `scripts/generate_all_shelves.py` - 批量生成40个GLB模型
- `frontend/public/assets/shelf-models-metadata.json` - 模型元数据
- `ModelLibrary.vue` - P2页自定义货架弹窗、两层级联选择器

### 已修复问题

3. **外墙标语功能完整实现** - 已完成（2026-03-26）
   - **开发背景**: 原V7.0开发计划中的外墙标语功能因"无法保存标语移动操作"问题于2026-03-24冻结，本次重新启动开发并完整修复
   - **问题1**: 标语移动后无法保存位置
     - **根因**: `onMouseMove` 中存在两套移动逻辑，实际执行的第二套逻辑直接操作 `position`，没有调用 `moveWallSign` 更新 `offsetAlongWall`
     - **修复**: 重构 `onMouseMove`，统一调用 `moveSelectedObjects` 作为唯一移动入口，确保 `moveWallSign` 被正确调用
   - **问题2**: 对齐线吸附逻辑异常
     - **根因**: 重构后 `moveSelectedObjects` 使用 `applySnap`（基于中心点吸附），而非 `checkAlignmentLineSnap`（基于4条边吸附）
     - **修复**: 将 `moveSelectedObjects` 中的吸附调用改为 `checkAlignmentLineSnap`
   - **问题3**: 字体大小改变时标语高度变化
     - **根因**: `createWallSign` 中Y坐标计算依赖标语3D高度（`height * 5`），字体越大3D高度越大
     - **修复**: 添加 `signHeight` 配置项（默认150cm），Y坐标使用 `baseHeight + signHeightConfig`，与字体大小解耦
   - **问题4**: 移动时标语高度变化
     - **根因**: `moveWallSign` 中 `sign.position.y += delta.y` 应用鼠标Y方向移动
     - **修复**: 移动时保持当前Y坐标不变（`sign.position.y = currentY`），高度改为通过编辑功能调整
   - **问题5**: 标语无法被选中
     - **根因1**: `objectInfo` 中未包含外墙标语特殊字段
     - **修复1**: 在 `selectObject` 中添加外墙标语字段（`text`, `fontSize`, `signHeight` 等）
     - **根因2**: 点击检测逻辑只优先处理门/窗，未处理外墙标语
     - **修复2**: 在点击检测优先逻辑中添加 `wallSign` 类型
   - **问题6**: 某些视角下标语不可见
     - **根因**: 标语距离墙体太近（5cm），`polygonOffset` 值太小（-1），在某些角度被墙体遮挡
     - **修复**: 向外偏移从5cm→10cm，`polygonOffsetFactor` 从-1→-10，添加 `polygonOffsetUnits: -10`
   - **功能增强**:
     - 添加标语高度编辑功能（范围0.5M-15M）
     - 隐藏外墙标语旋转按钮（暂时不需要旋转功能）
   - **相关代码**:
     - `ThreeScene.vue`: `createWallSign()`, `updateWallSign()`, `moveWallSign()`, `moveSelectedObjects()`, `selectObject()`, 点击检测逻辑
     - `CoreFunction.vue`: 编辑弹窗UI、保存/导入逻辑、旋转按钮隐藏

1. **门/窗保存功能** - 已修复（2026-03-25）
   - **问题**: 门/窗删除、添加操作后保存，重新导入时状态未正确恢复
   - **根因**: 保存逻辑中未对 `userData` 进行深层 `toRaw` 处理，导致 Vue 响应式 Proxy 干扰；导入过滤逻辑缺少容错性
   - **修复方案**:
     - 保存时使用 `toRaw` 解除 `obj` 和 `obj.userData` 的响应式
     - 添加 `businessType` 备用字段，防止 `type` 属性被覆盖
     - 导入时使用容错性过滤（`toLowerCase().trim()`）和诊断日志
   - **相关代码**: `CoreFunction.vue` 中的 `exportReport()` 保存逻辑和 `importProject()` 导入逻辑

2. **立柱功能完整修复** - 已修复（2026-03-25）
   - **问题1**: 立柱拖拽放置后保存不成功，重新导入后消失

     - 保存逻辑中未包含 `pillar` 类型过滤
     - 保存时从 `userData.position` 读取位置，但实际位置存储在 `obj.position`
     - `selectedObject.value` 是信息对象而非真正的 Three.js 对象，传递给 `updatePillarHeight` 导致 `userData` 访问失败
   - **修复方案**:
     - 在保存 filter 中添加 `type === 'pillar'` 判断
     - 保存时从 `obj.position` 读取位置，导入时转换为 `{x, z}` 格式
     - 在 `getSceneObjects()` 中添加 pillar 类型支持
     - 在 `confirmPillarHeightEdit()` 中通过 UUID 查找真正的 Three.js 对象
   - **相关代码**:
     - `CoreFunction.vue`: `confirmSaveProject()` 保存逻辑、`loadImportedObjects()` 导入逻辑、`confirmPillarHeightEdit()` 编辑确认逻辑
     - `ThreeScene.vue`: `getSceneObjects()` 对象获取逻辑

### 下一步计划
1. **撤销/重做功能** - 操作历史管理
2. **框选多对象** - 批量选择功能

### 技术债务（后续版本优化）

#### 中优先级
1. **THREE dispose 资源释放**
   - **问题**: ThreeScene.vue 中模型创建函数（createShelfModel, createContainerModel 等）创建的大量几何体和材质没有对应的 dispose 逻辑
   - **影响**: 当模型从场景中移除时，可能造成内存泄漏
   - **建议修复**: 添加 disposeModel() 辅助函数，在移除对象时遍历释放 geometry 和 material

#### 低优先级
2. **Vector3 对象池（性能优化）**
   - **问题**: 代码中大量 `new THREE.Vector3()` 创建临时对象，增加 GC 压力
   - **影响**: 频繁创建/销毁 Vector3 对象可能影响性能
   - **建议修复**: 使用对象池模式，复用 Vector3 实例减少 GC

1. `mousedown` → `mouseup` → `click` 事件顺序
2. `onTextLabelMouseDown` 中启动拖动，使用 `window.addEventListener('mouseup', onMouseUp)`
3. `onCanvasClick` 使用 Vue 的 `@click`，与 `window` 上的 `mouseup` 是独立的监听机制

**修复方案：标志位拦截**
```javascript
// 添加标志位
const ignoreNextCanvasClick = ref(false);

// 文字标注 mousedown 处理
function onTextLabelMouseDown(event, labelId) {
  ignoreNextCanvasClick.value = true;  // 标记：接下来忽略画布click
  selectedTextLabel.value = labelId;
  // ... 其他逻辑
}

// 画布 click 处理
function onCanvasClick(event) {
  if (ignoreNextCanvasClick.value) {
    ignoreNextCanvasClick.value = false;  // 重置标志
    return;  // 不执行清空选中
  }
  // ... 原有逻辑
}
```

### 2026-03-16: 批量复制旋转后货架重叠Bug修复

**问题描述**
货架旋转90度后，进行批量复制（如3行2列，行距3米，列距0），中型、轻型货架出现重叠，而高位货架正常。

**根因分析**
批量复制位置计算使用了**世界坐标系尺寸**：
```javascript
const rowOffset = r * (getObjectDepth(originalObj) + rowSpacing * 100);
const colOffset = c * (getObjectWidth(originalObj) + colSpacing * 100);
```

当货架旋转90度后：
- 世界坐标系的X尺寸 = 原货架的Z尺寸（长度）
- 世界坐标系的Z尺寸 = 原货架的X尺寸（宽度）

导致行/列间距计算错误，货架重叠。

**修复方案：使用本地尺寸**

**1. 新增获取本地尺寸的方法**（不考虑旋转）：
```javascript
// 获取对象本地宽度（模型本身的X尺寸）
function getObjectLocalWidth(obj) {
  const clonedObj = obj.clone();
  clonedObj.rotation.set(0, 0, 0);
  const box = new THREE.Box3().setFromObject(clonedObj);
  return box.max.x - box.min.x;
}

// 获取对象本地深度（模型本身的Z尺寸）
function getObjectLocalDepth(obj) {
  const clonedObj = obj.clone();
  clonedObj.rotation.set(0, 0, 0);
  const box = new THREE.Box3().setFromObject(clonedObj);
  return box.max.z - box.min.z;
}
```

**2. 修改批量复制位置计算**：
```javascript
// 使用本地尺寸，不受旋转影响
const rowOffset = r * (getObjectLocalDepth(originalObj) + rowSpacing * 100);
const colOffset = c * (getObjectLocalWidth(originalObj) + colSpacing * 100);
```

**修复后行为**
- 无论货架旋转多少度（0°、90°、180°、270°），批量复制都使用货架的**真实本地尺寸**
- 行间距基于货架本地深度（前后方向）
- 列间距基于货架本地宽度（左右方向）
- 轻型、中型、高位货架旋转后都能正确排列，不会重叠

**影响文件**
- `frontend/src/components/3d/ThreeScene.vue` - 批量复制位置计算逻辑、新增本地尺寸获取方法

### 2026-03-24: 对象移动Y坐标修复

**问题描述**

1. 直接拖拽移动时：`firstObj.position.copy(targetPosition)` 将Y坐标设为0

**修复方案**
1. 直接拖拽：只更新X和Z坐标，保持Y坐标不变
   ```javascript
   firstObj.position.x = targetPosition.x;
   firstObj.position.z = targetPosition.z;
   ```
2. 吸附移动：强制 `delta.y = 0`

**代码修改**
- `ThreeScene.vue` 第2967行：添加 `delta.y = 0;`
- `ThreeScene.vue` 第3030-3031行：改为只更新X和Z坐标

### 2026-03-24: 3D网格坐标轴单位统一修复

**问题描述**
3D画布的网格坐标轴单位不统一，每次绘制不同大小的仓库时，网格的显示比例会变化，导致每个正方形网格的实际边长不一致。用户无法通过网格直观判断距离。

**根本原因**
网格生成逻辑使用 `gridDivisions = Math.floor(gridSize / 10)`，根据仓库尺寸动态计算网格划分，导致不同大小仓库的网格视觉尺寸不同。

**修复方案**
1. **固定网格参数**：
   - 网格单元格固定为1米（100cm）
   - 网格总范围固定为100米×100米（10000cm×10000cm）
   - 网格划分固定为100×100个单元格

2. **添加坐标轴刻度**：
   - 每1米（100cm）显示一个短刻度标记
   - 每10米（1000cm）显示一个长刻度标记+数字标签（如"10m"）

3. **网格中心对齐**：
   - 网格中心始终与仓库中心对齐
   - 透明度保持10%

**代码修改**
- `ThreeScene.vue`:
  - 添加常量：`GRID_CELL_SIZE = 100`, `GRID_TOTAL_SIZE = 10000`, `GRID_DIVISIONS = 100`
  - 修改 `createWarehouse()` 网格生成（Line 4419-4426）
  - 修改 `createWarehouse()` 网格辅助线更新（Line 4493-4502）
  - 修改 `createWarehouseFromShape()` 网格生成（Line 4554-4563）
  - 修改 `createWarehouseFromShape()` 网格辅助线更新（Line 4628-4637）
  - 修改 `createDirectionLabels()` 添加刻度标记和标签（Line 2389-2466）
  - 修改 `updateDirectionLabels()` 添加刻度标记和标签（Line 2581-2660）

**用户价值**
- 通过网格直观判断1米距离
- 放置对象时更有空间感
- 使用测量工具验证网格边长显示1.00米
